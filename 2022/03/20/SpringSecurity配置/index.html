Spring Security 配置

1.@EnableWebSecurity 注解

@EnableWebSecurity 导入了4个配置类

  • WebSecurityConfiguration.class,
  • SpringWebMvcImportSelector.class,
  • OAuth2ImportSelector.class,
  • HttpSecurityConfiguration.class

包含@EnableGlobalAuthentication注解导入了AuthenticationConfiguration.class配置类，

AuthenticationConfiguration配置类配置了AuthenticationManager(认证管理器)。

2.WebSecurityConfiguration 配置类

主要源码

这个类里面比较重要的就两个方法：springSecurityFilterChain和setFilterChainProxySecurityConfigurer。

setFilterChainProxySecurityConfigurer 配置WebSecurity

Sets the {@code <SecurityConfigurer<FilterChainProxy, WebSecurityBuilder>} instances used to create the web configuration.

  • ObjectPostProcessor<Object> objectPostProcessor 创建实例并增强
    • 负责创建增强WebSecurity实例
  • @Value(“#{@autowiredWebSecurityConfigurersIgnoreParents.getWebSecurityConfigurers()}”) List<SecurityConfigurer<Filter, WebSecurity>> webSecurityConfigurers 获取所有注入spring容器的SecurityConfigurer实现的配置类
    • 获取到了我们自定义的继承WebSecurityConfigurerAdapter的配置类
    • AutowiredWebSecurityConfigurersIgnoreParents类

这个方法主要是实例化WebSecurity 并将 SecurityConfigurer接口实现的配置类添加到WebSecurity中

springSecurityFilterChain

通过WebSecurity构建FilterChainProxyl类

3. WebSecurity.build() 构建FilterChainProxyl类

build方法是由抽象父类的父类实现的。

AbstractSecurityBuilder.build()

AbstractConfiguredSecurityBuilder.doBuild()方法执行构造主要逻辑

只有init() 和 performBuild() 主要执行了构造逻辑，其他方法都是空实现。

WebSecurityConfigurerAdapter.init()

WebSecurity.performBuild() 具体实现构建逻辑

HttpSecurity.build()方法

和WebSecurity构建流程一样，不过WebSecurity中保存的配置类只有我们自定义继承WebSecurityConfigurerAdapter类。在调用**WebSecurity.init()**方法中实例化了HttpSecurity并为其设置了默认配置，而我们继承的WebSecurityConfigurerAdapter重写的configure(HttpSecurity http) 等方法将与默认配置一起作用。
